I need you to build a full-stack Webhook Testing & Tunneling application called "HookTest".

**Architecture:** Monolithic (Node.js/Express backend serving a React/Vite frontend).
**Goal:** A user generates a unique webhook URL. When that URL is hit, the request appears in real-time on the Dashboard AND is forwarded to the user's localhost via a CLI script.

Please create the following file structure and code.

### 1. Project Structure
- Root: `server.js`, `cli.js`, `package.json`
- Client: `client/` (Vite + React + Tailwind)

### 2. Root `package.json`
- Dependencies: `express`, `socket.io`, `ioredis`, `concurrently`, `cors`, `dotenv`.
- DevDependencies: `nodemon`.
- Script "dev": `concurrently "nodemon server.js" "cd client && npm run dev"`
- Script "start": `node server.js`
- Script "build": `cd client && npm install && npm run build`

### 3. Backend Logic (`server.js`)
- Initialize Express and Socket.io (allow CORS for MVP).
- Create a `Map` or use Redis for storing webhook history.
- Route `POST /api/new`: Generate a UUID and return `{ uuid, url }`.
- Route `ALL /webhook/:uuid`:
  1. Store request details (headers, body, method) in history.
  2. Emit event `new-request` to room `dashboard:{uuid}`.
  3. Emit event `tunnel-request` to room `tunnel:{uuid}`.
  4. Send 200 OK.
- Socket.io:
  - On `join-dashboard`: join room `dashboard:{uuid}`.
  - On `register-tunnel`: join room `tunnel:{uuid}`.
- Production Serve: Check if `process.env.NODE_ENV === 'production'`, if so, serve `client/dist`.

### 4. CLI Agent (`cli.js`)
- Use `socket.io-client` and `axios`.
- Usage: `node cli.js <UUID> <LOCAL_PORT>`.
- Connect to the server via Socket.io.
- On `connect`: emit `register-tunnel` with the UUID.
- On `tunnel-request`: Use Axios to forward the request to `http://localhost:LOCAL_PORT`.
- Log success/failure to console.

### 5. Frontend (`client/`)
- Setup: React + Vite + Tailwind CSS.
- **Vite Config:** Configure `server.proxy` to forward `/api` and `/socket.io` to `http://localhost:3000`.
- **App.jsx:**
  - On load: Call `/api/new` to get a session if one doesn't exist.
  - Connect Socket.io client.
  - Sidebar: List incoming requests (listen for `new-request`).
  - Main View: Show JSON details of selected request (Headers/Body).
  - Show the "CLI Command" to the user: `node cli.js <UUID> 3000`.

**Execution Plan:**
1. Scaffold the folders.
2. Write the backend code.
3. Write the CLI code.
4. Setup the React client with Tailwind.
5. Install all dependencies for root and client.
